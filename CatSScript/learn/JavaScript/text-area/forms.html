HTML Forms

An HTML form is used to collect user input. The user input is most often sent to a server for processing.

The <form> Element

The HTML <form> element is used to create an HTML form for user input.
The syntax for the element is the following:

<form action="action to do" method="POST or GET" name="form name" onsubmit="return validityFunction()">
form elements
</form>

The <form> element is a container for different types of input elements, such as: text fields, checkboxes, radio buttons, submit buttons, etc.

HTML Form Elements

These are some of the form elements that the HTML <form> element can contain:

  • <input>
  • <select>
  • <textarea>
  • <datalist>

The <input> Element

One of the most used form element is the <input> element.
The <input> element can be displayed in several ways, depending on the type attribute.

These are some of the different values of the type attribute (the default value of the type attribute is "text":

  • <input type="text">:
    This type of input defines a single-line text input field.
    For example, if you have the following HTML code: 
    1. <form>
    2.   First name:<br>
    3.   <input type="text" id="fname" name="fname"><br>
    4.   Last name:<br>
    5.   <input type="text" id="lname" name="lname">
    6. </form>
    You will actually see this in the browser:
    First name:

    Last name:

  • <input type="password"> This type of input defines a password field, this means that if you write something into this type of field in a form, everything will be masked by asterisks or circles.
    For example, if you have the following HTML code: 
    1. <form>
    2.   Username:<br>
    3.   <input type="text" id="username" name="username"><br>
    4.   Password:<br>
    5.   <input type="password" id="pwd" name="pwd">
    6. </form>
    You will actually see this in the browser:
    Username:

    Password:

  • <input type="submit"> This type of input defines a submit button form data to a form-handler.
    The form-handler is typically a server page with a script for processing input data.
    The form-handler is specified in the form's <action> attribute.
    For example, if you have the following HTML code: 
    1. <form action="">
    2.   First name:<br>
    3.   <input type="text" id="fname" name="fname" value="Mario"><br>
    4.   Last name:<br>
    5.   <input type="text" id="lname" name="lname" value="Rossi"><br><br>
    6.   <input type="submit" value="Submit">
    7. </form>
    You will actually see this in the browser:
    First name:

    Last name:



    Note that in the following example only the submit button is shown, this means that no action is linked to the form.

  • <input type="reset">
    This type of input defines a reset button that will reset all form values to their default values:
    For example, if you have the following HTML code: 
    1. <form action="">
    2.   First name:<br>
    3.   <input type="text" id="fname" name="fname" value="Mario"><br>
    4.   Last name:<br>
    5.   <input type="text" id="lname" name="lname" value="Rossi"><br><br>
    6.   <input type="submit" value="Submit">
    7.   <input type="reset" value="Reset">
    8. </form>
    You will actually see this in the browser:
    First name:

    Last name:



    If you change the input values and then click the "Reset" button, the form-data will be reset to the default values.
  • <input type="radio">
    This type of input defines a radio button.
    Radio buttons let a user select only one of a limited number of choices.
    For example, if you have the following HTML code: 
    1. <form>
    2.   <input type="radio" id="male" name="gender" value="male">
    3.   Male<br>
    4.   <input type="radio" id="female" name="gender" value="female">
    5.   Female<br>
    6.   <input type="radio" id="other" name="gender" value="other">
    7.   Other
    8. </form>
    You will actually see this in the browser:
    Male
    Female
    Other

    Note that the checked attribute specfifies that an input field should be pre-selected when the page loads.
  • <input type="checkbox">
    This type of input defines a checkbox.
    Checkboxes let a user select zero or more options of a limited number of choices.
    For example if you have the following HTML code: 
    1. <form>
    2.   <input type="checkbox" id="vehicle1" name="vehicle1" value="Bike">
    3.   I have a bike<br>
    4.   <input type="checkbox" id="vehicle2" name="vehicle2" value="Car">
    5.    I have a car<br>
    6.   <input type="checkbox" id="vehicle3" name="vehicle3" value="Boat">
    7.    I have a boat
    8. </form>
    You will actually see this in the browser:
    I have a bike
    I have a car
    I have a boat

    Note that the checked attribute specfifies that an input field should be pre-selected when the page loads.
  • <input type="date">
    This type of input is used for fields that should contain a date.
    A date picker can show up in the input field.
    For example if you have the following HTML code: 
    1. <form>
    2.   Birthday:
    3.   <input type="date" id="birthday" name="birthday">
    4. </form>
    You will actually see this in the browser:
    Birthday:

  • <input type="email">
    This type of input is used for fields that should contain an e-mail addess.
    This type of input provides a static validation for the e-mail address.
    For example, if you have the following HTML code: 
    1. <form>
    2.   Enter your email:
    3.   <input type="email" id="email" name="email">
    4. </form>
    You will actually see this in the browser:
    Enter your email:

  • <input type="file">
    This type of input defines a file-select field and a "Browse" button for file uploads.
    For example, if you have the following HTML code: 
    1. <form>
    2.   Select a file:
    3.   <input type="file" id="myfile" name="myfile">
    4. </form>
    You will actually see this in the browser:
    Select a file:

  • <input type="number">
    This type of input defines a numeric input field.
    You can also set restriction on what numbers are accepted with min and/or max attributes.
    For example, if you have the following HTML code: 
    1. <form>
    2.   Quantity (between 1 and 5):
    3.   <input type="number" id="quantity" name="quantity" min="1" max="5">
    4. </form>
    You will actually see this in the browser:
    Quantity (between 1 and 5):

The <select> Element

The <select> element defines a drop-down list.
The <option> elements defines an option that can be selected.
By default the first item in the drop-down list is selected; to define a pre-selected option, add the selected attribute to the option.
For example, the following HTML code: 

  1. Choose a car:
  2. <select id="cars" name="cars">
  3.   <option value="volvo" selected>Volvo</option>
  4.   <option value="saab">Saab</option>
  5.   <option value="fiat">Fiat</option>
  6.   <option value="audi">Audi</option>
  7. </select>

will produce:
Choose a car:


The <textarea> Element

The <textarea> element defines a multi-line input field.
The rows attribute specifies the visible numbers of lines in a text area.
The cols attribute specifies the visible width of a text-area.
For example this HTML code:

The <datalist> Element

The <datalist> element specifies a list of pre-defined options for an <input> element.
Users will see a drop-down list of the pre-defined options as they input data.
The list attribute of the <input> element, must refer to the id attribute of the <datalist> element.
For example, this HTML code: 

  1. <form action="">
  2.   <input list="browsers">
  3.   <datalist id="browsers">
  4.     <option value="Internet Explorer">
  5.     <option value="Firefox">
  6.     <option value="Chrome">
  7.     <option value="Opera">
  8.     <option value="Safari">
  9.   </datalist>
  10. </form>
